## 服务端部署

### 环境要求
- Node.js >= 14.0.0
- npm >= 6.0.0

### 本地部署步骤

#### 1. 安装依赖

```bash
cd server
npm install
```

#### 2. 启动服务

**开发模式**:
```bash
npm run dev
```

**生产模式**:
```bash
npm start
```

#### 3. 验证服务

访问 `http://localhost:3000/api` 应该返回：
```json
{
  "message": "守护星服务端运行中",
  "version": "1.0.0"
}
```

### 使用 PM2 部署（推荐）

#### 安装 PM2

```bash
npm install -g pm2
```

#### 启动应用

```bash
pm2 start server.js --name guardian-star
```

#### 常用命令

```bash
# 查看状态
pm2 status

# 查看日志
pm2 logs guardian-star

# 重启应用
pm2 restart guardian-star

# 停止应用
pm2 stop guardian-star

# 开机自启
pm2 startup
pm2 save
```

### Docker 部署

#### 创建 Dockerfile

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --production
COPY . .
EXPOSE 3000 3001
CMD ["node", "server.js"]
```

#### 构建并运行

```bash
docker build -t guardian-star .
docker run -d -p 3000:3000 -p 3001:3001 --name guardian-star guardian-star
```

## Android应用构建

### 环境要求
- JDK 17
- Android SDK
- Gradle 8.2

### 家长端构建

#### 1. 进入项目目录

```bash
cd parent
```

#### 2. 配置签名（可选）

在 `app/build.gradle` 中配置签名：

```gradle
android {
    signingConfigs {
        release {
            storeFile file("your-keystore.jks")
            storePassword "your-password"
            keyAlias "your-alias"
            keyPassword "your-key-password"
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
        }
    }
}
```

#### 3. 构建 APK

**Debug版本**:
```bash
./gradlew assembleDebug
```

**Release版本**:
```bash
./gradlew assembleRelease
```

构建完成后，APK文件位于：
```
app/build/outputs/apk/debug/app-debug.apk
app/build/outputs/apk/release/app-release.apk
```

### 儿童端构建

步骤同上，进入 `child` 目录执行相应命令。

## 生产环境配置

### 1. 网络配置

#### 端口开放

确保以下端口需要对外开放：
- **3000**: HTTP API
- **3001**: WebSocket

#### 防火墙配置（Ubuntu）

```bash
# 开放端口
sudo ufw allow 3000/tcp
sudo ufw allow 3001/tcp

# 查看状态
sudo ufw status
```

### 2. Nginx 反向代理配置

#### 安装 Nginx

```bash
sudo apt update
sudo apt install nginx
```

#### 配置文件

创建 `/etc/nginx/sites-available/guardian-star`:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }

    location /ws/ {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }
}
```

#### 启用配置

```bash
sudo ln -s /etc/nginx/sites-available/guardian-star /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 3. HTTPS 配置（推荐）

使用 Let's Encrypt 免费证书：

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

### 4. 数据库备份

#### 定时备份脚本

```bash
#!/bin/bash
BACKUP_DIR="/var/backups/guardian-star"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
cp /path/to/guardian_star.db $BACKUP_DIR/guardian_star_$DATE.db

# 保留最近7天的备份
find $BACKUP_DIR -name "guardian_star_*.db" -mtime +7 -delete
```

#### 添加到 crontab

```bash
crontab -e
```

添加以下内容（每天凌晨2点备份）：
```
0 2 * * * /path/to/backup-script.sh
```

### 5. 环境变量配置

创建 `.env` 文件：

```env
PORT=3000
WS_PORT=3001
NODE_ENV=production
```

修改 `server.js` 读取环境变量：

```javascript
require('dotenv').config();
const PORT = process.env.PORT || 3000;
```

## 故障排查

### 常见问题

#### 1. WebSocket 连接失败

**检查项：**
- 服务是否正常运行
- 端口是否开放
- 防火墙配置是否正确
- 客户端连接地址是否正确

**调试命令：**
```bash
# 检查端口监听
netstat -tlnp | grep 3001

# 测试连接
curl http://localhost:3000/api
```

#### 2. Android 应用连接失败

**检查项：**
- 设备网络连接正常
- 服务端IP地址配置正确
- 应用内权限已授予
- 后台服务未被系统清理

#### 3. 数据库错误

**查看数据库文件权限：**
```bash
# 检查数据库文件权限
ls -l guardian_star.db

# 修复权限
chmod 644 guardian_star.db
```

## 安全建议

1. **使用 HTTPS**: 生产环境必须启用 HTTPS
2. **API 认证**: 考虑添加 API 密钥认证
3. **数据库加密**: 敏感数据进行加密存储
4. **定期备份**: 配置定期数据备份
5. **访问控制**: 限制管理后台访问IP
6. **日志审计**: 定期检查访问日志
